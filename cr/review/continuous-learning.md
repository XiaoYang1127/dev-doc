# 持续学习

基于反馈闭环，持续优化审核质量。

---

## 定位

AI 审核的痛点：AI 认为有问题，但人工认为没问题（误报），或者 AI 漏掉了问题。

持续学习的目标是：让 AI 学会团队的审核标准，减少误报和漏报。

---

## 学习方式

### 1. 反馈闭环

```
审核意见发出
    │
    ▼
开发者反馈（接受/拒绝/修改）
    │
    ▼
反馈记录存储
    │
    ▼
定期分析反馈模式
    │
    ▼
更新审核规则/Skill
```

### 2. 审核小助手

在 GitLab 上增加「审核小助手」机器人账号：

| 功能 | 说明 |
|------|------|
| **学习人工审核** | 观察人类审核者的意见 |
| **记录反馈** | 记录每条审核意见的处理结果 |
| **自我优化** | 基于反馈调整审核标准 |

### 3. Skill 迭代

基于反馈持续打磨审核要求：

| 阶段 | 输入 | 输出 |
|------|------|------|
| 人工打磨 | 反馈记录 | 更新 `feedback/review-log.md` |
| 规则优化 | 反馈分析 | 更新 `rules/rule-framework.md` |
| Skill 更新 | 完整反馈 | 发布新版本 Skill |
