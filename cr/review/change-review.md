# 变更审核

针对 MR / 代码变更的实时审核，是最核心的审核场景。

---

## 审核层次

### L1: 变更感知

获取代码变更信息，包括：
- 变更文件列表
- 变更行数
- 变更差异（diff）

### L2: 片段审核

**只审核变更本身**：
- 写法是否有问题
- 设计与实现是否合理

### L3: 上下文审核

**结合变更上下文审核**：
- 变更行的上下几行
- 变更所在的完整函数/方法/类
- 追加：变更上下文逻辑是否有问题

### L4: 架构审核

**结合代码调用关系审核**：
- 分层越界：Controller 直接操作 Repository
- PO 直接返回前端
- 循环依赖

### L5: 调用链审核（待实现）

**分析上下游调用关系**：
- 某一处的变更是否影响全局
- 是否符合设计模式

---

## 当前检查项

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| 分层越界 | Controller 直接操作 Repository、PO 直接返回前端等 | P0 |
| 事务安全 | @Transactional 内的远程调用/异步操作 | P0 |
| NPE 风险 | 可能空指针的场景 | P1 |
| 明显逻辑缺陷 | 死代码、资源未关闭、错误比较等 | P1 |

**暂不检查**：代码风格、注释、魔法数字、方法长度、圈复杂度等

---

## 审核流程

```
代码提交 / MR 创建
       │
       ▼
获取变更信息（L1）
       │
       ▼
片段审核（L2）
       │
       ▼
上下文审核（L3）
       │
       ▼
架构审核（L4）
       │
       ▼
生成审核意见
       │
       ▼
返回结果 / MR 评论
```
