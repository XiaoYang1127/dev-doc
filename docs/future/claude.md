# 讨论模式

## 适用场景

基于参考资料（reference/）、背景材料（如 0401.md），整理成体系的管理思考。

## 讨论模式

### 1. 明确话题

先确认要整理哪些方向，排出优先级，一次专注一个方向深入。

### 2. 讨论进程追踪

每个话题讨论时，记录当前进度：
- 当前讨论到哪个子话题
- 已确认的判断/结论是什么
- 待确认或待补充的是什么

### 3. 聚焦提问

每个话题问 1-2 个聚焦的问题，引导到管理判断层面。问题要具体，避免太开放。

### 4. 核心判断优先

用户给出核心观点/结论后，再展开细节。先确认判断正确，再补充细节和案例。

### 5. 迭代优化

- 措辞不准确 → 直接给选项让用户选
- 表达太泛 → 要求举团队例子
- 逻辑跳跃 → 指出哪里不顺，问要不要改

### 6. 成文后通读

全文完成后，逐段审视：
- 上下文衔接是否自然
- 措辞是否贴近团队语境
- 有没有假大空

### 7. 敏态调整

讨论过程中，用户可以随时喊停调整。比如：
- 「XX措辞更好」
- 「XX太局限了」
- 「这个表达不够贴近研发团队」

## 记录习惯

### 参考材料

读过的材料，在讨论开头梳理摘要，帮助对齐上下文。

### 关键判断

用户的核心观点单独记录，不要散落在讨论里。方便最后整理时抓住主干。

### 讨论结果

每个话题的结论明确记下来，避免最后整理时丢失。
