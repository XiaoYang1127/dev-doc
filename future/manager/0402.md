## 04-02 讨论结论

### 秩序层面

**问题诊断：表面遵守、本质敷衍**

- 14/16（建工单/更新工单）被遗忘 → 缺少反馈闭环
- 18（计划延期）被执行 → 因为有实质处罚
- **反馈闭环未打通**：填了没人看、做了没反馈 → 没动力做实

**三个卡点：**

- 负责人精力不够 — 想盯但盯不过来
- 缺少自动汇总 — 信息分散，没人整理
- 缺少公开追踪 — 做得好坏团队看不到

**解法：方向一 + 方向三组合（工具自动化 + 聚焦关键规约）**

**规约 14/16 强化：**

| 项目         | 内容                                       |
| ------------ | ------------------------------------------ |
| 工单内容标准 | 必须包含：执行思路、时间安排、会议结论     |
| 触发机制     | 到期未更新 → 自动提醒 → 3天未改 → 延期记录 |
| 汇总曝光     | 每周固定曝光延期工单列表（专人负责）       |
| 处罚兑现     | 每月兑现，对齐 18 的标准（扣绩效 + 罚钱）  |

**专人专责：**

- 人选：相对清闲的成员，职责写入正式定义
- 核心职责：汇总工单状态 → 每周形成延期报告 → 在计划会上通报
- 执行逻辑：让数据说话，减少负责人手动判定成本

---

### 效率层面

**问题诊断：什么都等少数人拍板，评审响应慢，来回往返多**

- 判断的事太多 → 什么都等少数人判断 → 判断者成为瓶颈
- 来回往返多 → 提交材料不完整、方向性有问题
- 响应慢 → 判断者事情太多、切换成本高

**解法：模板 + AI 门禁 + 分层判断**

| 问题         | 解法                                   |
| ------------ | -------------------------------------- |
| 判断的事太多 | 分层——AI 检查通过才能提人工评审        |
| 来回往返多   | 提交前模板自检，AI 检查完整性 + 方向性 |
| 响应速度慢   | 人工只判方向，不补材料                 |

**AI 门禁机制：**

- 模板放代码仓库根目录
- 规则由技术负责人维护
- 提交前 AI 按模板检查 → 通过才进入人工评审
- 检查项：完整性（材料是否齐全）+ 方向性（思路对不对）
