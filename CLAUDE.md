# Claude Code 项目规范

---

## 项目文档模板

每个项目文档按以下顺序组织：

```
1. 定位与目标
2. 架构设计
3. 边界职责
4. 流程设计
5. TODO
```

### 定位与目标

回答三个问题：

| 问题           | 说明               |
| -------------- | ------------------ |
| 做什么的？     | 一句话描述核心功能 |
| 解决什么问题？ | 痛点和价值         |
| 目标是什么？   | 可量化的目标       |

### 架构设计

| 内容           | 说明                     |
| -------------- | ------------------------ |
| 整体架构图     | 模块组成 + 模块间关系    |
| 模块边界与依赖 | 每个模块的职责和依赖关系 |
| 设计原则       | 核心设计原则             |

### 边界职责

每个模块定义：

```
模块 A
├── 职责：XXX
├── 不管：XXX
├── 被谁调用：XXX
└── 调用谁：XXX
```

### 流程设计

核心流程覆盖主要场景：

```
触发阶段 → 执行阶段 → 结果阶段 → 回执阶段
```

必问：

- 流程是否闭环？
- 失败如何处理？
- 状态如何流转？

### TODO 管理

有价值但当前不紧急的内容：

```markdown
| 事项 | 优先级   | 说明 |
| ---- | -------- | ---- |
| XXX  | 高/中/低 | XXX  |
```

## 目录划分与命名

**原则**：自然好读、层级浅

**保留原始想法**：每个目录下可保留 `z_note.md` 记录原始思考过程，方便回溯时不丢失灵感。不要删除，事后可整理到正式文档中。

| 问题     | 错误                            | 正确                     |
| -------- | ------------------------------- | ------------------------ |
| 路径过深 | `docs/superpowers/specs/xxx.md` | `xxx-design.md`          |
| 命名冗余 | `report/report.md`              | `reference/benchmark.md` |
| 分类模糊 | `unit/`（只放 java 单测）       | `guides/`                |
